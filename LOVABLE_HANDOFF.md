## 2. Content Structure (The Curriculum)
The application should be structured around these 11 Chapters, grouped into 4 Phases (matching the panels):

### Phase 1: Contract & Setup (Panel 1)
*   **Chapter 1:** Awarded Contract/Purchase Order (Review, Compare to Bid, Scope Review)
*   **Chapter 2:** Execute Contract (Signatures)
*   **Chapter 3:** Start-up Documents (COI, SOV, Misc Docs)

### Phase 2: System Setup & Approvals (Panel 2)
*   **Chapter 4:** System Input (Trello, QuickBooks, Paper Organization)
*   **Chapter 5:** Submittals (Request Quotes, Approve Quotes, Submit Drawings)
*   **Chapter 6:** Release to Production (Approval Received, Lead Times, Drop Ship)

### Phase 3: Installation & Invoicing (Panel 3)
*   **Chapter 7:** Coordinate Install (Installer Quotes, Scheduling)
*   **Chapter 8:** Final Install (Confirm Delivery, Confirm Date, Complete Install)
*   **Chapter 9:** Invoicing (Pay Apps G702/G703, Final Retainage Bill)

### Phase 4: Closeout (Panel 4)
*   **Chapter 10:** Close-Out Docs (Warranties, Lien Waivers, Punch List)
*   **Chapter 11:** Final Payment (Receive Payment, Pay Vendors, Archive Project)

## 3. Design Specifications
*   **Primary Color:** Moon River Blue (`#1e40af` or similar deep blue)
*   **Background Style:** Clean, professional, using the "Blueprint Grid" motif where appropriate.
*   **Typography:** Clean sans-serif (Inter or Roboto) for readability.
*   **Vibe:** Educational, structured, professional, yet engaging ("Gamified" progress tracking is a plus).
